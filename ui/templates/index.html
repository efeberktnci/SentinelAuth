SentinelAuth

SentinelAuth is a local proof-of-concept that demonstrates identity tampering in an OpenID-style callback flow. The project includes a vulnerable login path, a secure login path, and a minimal manual test environment for reproducing the attack with Burp Suite.

Open Vulnerable Login Open Secure Login

Threat Model

The vulnerable pattern appears when a server accepts a claimed identity from a callback request without verifying that the signature still belongs to that claimed identity. An attacker can authenticate as one account, tamper with the callback, and attempt to assume another user's session.

1. Login begins User authenticates with valid credentials.
2. Callback is signed Server binds the original identity to an HMAC.
3. Request is intercepted Attacker modifies the identity fields before the callback reaches the server.
4. Outcome depends on validation Vulnerable flow accepts the tampered claim; secure flow blocks it.

Validation Paths

Vulnerable Path
Trusts claimed identity directly
Secure Path
Recomputes HMAC for claimed identity
Primary Risk
Account takeover through identity tampering
Mitigation
Integrity validation with HMAC-SHA256

Manual Reproduction

Use Burp Suite to log in as the lower-privilege account, intercept the callback, modify the identity fields, and compare the result between the vulnerable and secure login pages. The same tampering attempt should succeed only in the vulnerable flow.
